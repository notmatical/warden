# @warden/desktop

## 0.1.12

### Patch Changes

- ab56997: Fixed AskUserQuestion in chat sessions: it no longer shows a spurious "permission needed" prompt, and the working spinner now stops while the agent waits for your reply.
- d6c6180: Stop guessing a feature/ prefix for issue worktree branches: Linear issues use the workspace-configured branch name from the Linear API, GitHub issues follow GitHub's number-title-slug convention, and the branch is editable in the send-to-agent dialog
- ccb6886: Pre-fill the worktree branch name from the issue when sending a Linear or GitHub issue to an agent (feature/WAR-123, feature/issue-42)
- 6c8f618: Fix Codex turn-level errors hanging a session (and any workflow run driving it). An invalid-model 400 like "model is not supported when using Codex with a ChatGPT account" arrives as a non-retrying error with no thread id; previously it was dropped and the turn ran forever. Now a bare top-level error is routed to the sole in-flight turn, a non-retrying error (absent `willRetry` included) ends the turn and settles the session to Error, and a failed `Result` settles to Error instead of Idle for every run-to-completion provider. A workflow node whose agent fails now fails the run.
- ab56997: The composer's model, mode, and effort pickers have a cleaner, more consistent look. Fast mode now appears as an inline toggle on the models that support it, rather than a separate switch at the bottom of the model picker.
- 11f00a6: Add a global command palette (Cmd/Ctrl+K): jump to sessions and folders, start agent or terminal sessions in any folder, peek Linear issues and send them to an agent, switch theme, and run keybound commands from one keyboard-first surface.
- 56e7a66: PR background polling no longer bumps a session's last-active time, so the sessions table's Last active column reflects real activity instead of the poll cadence.
- bc3c8d8: Add OpenCode as a third agent provider: agent sessions run over the local `opencode serve` HTTP API with live text streaming, tool-call rendering, and interactive permission/question prompts; native OpenCode terminals resume their exact conversation; plus managed CLI install/updates, auth status, and a model picker that lists only the models the signed-in OpenCode account can actually run.
- 77d5b80: Remove the "Review a PR" new-session option and its checkout-PR worktree flow. Open PRs still surface in the PR status table.
- 56e7a66: Surface pull request status in the folder view's sessions table: a PR column with state-tinted icon, draft/merged/closed/review pills, and per-state CI check tallies, all refreshed by the existing background poller. The open PRs section and the session header's PR chip pick up the same treatment.
- 56e7a66: Sessions untouched for two weeks now show an amber warning in the folder view's Last active column, suggesting they may no longer be needed. Pinned and running sessions are never flagged, and nothing is archived automatically.
- 77d5b80: Unify Tasks and Issues into a single Issues destination with a Linear/GitHub source switch, instead of two near-identical top-level tabs. The selected source persists, and old "tasks" tab references redirect to Issues.
- 77d5b80: Add an in-app MCP server (`warden mcp`) so agents can act on connected integrations during a session: create, comment on, and set the status of Linear issues, and comment on or open/close GitHub issues. On by default when Linear or the GitHub CLI is available, with a toggle on the Linear settings row.
- 5ce652f: Workflow editor hardening and panel polish: run status is scoped to the viewed workflow, duplicated/imported workflows get fresh node ids, pending edits flush when the tab closes, connections that would form a cycle are refused, a second concurrent run is blocked, canceled and restart-interrupted runs settle instead of showing as running forever, and the node side panel gains an identity header, status pills, error output, gate approval controls, and a Stop button on the canvas.
- 6c8f618: Workflows now flow left-to-right instead of top-to-bottom: node handles move to the left (in) and right (out) edges, new nodes lay out horizontally, and tall cards sit side by side instead of stacking and overlapping. A "Tidy layout" action arranges an existing workflow into clean dependency columns and refits the view.
- 5ce652f: Prompt-injection hardening for workflows: permission-mode overrides are only honored for Custom nodes (a stale override can no longer make a Review/Plan node writable), injected context carries an explicit data-not-instructions boundary, the injected diff uses a fence that embedded backticks cannot break out of, and the Custom mode menu shows the real default (acceptEdits).
- 6c8f618: The workflow editor's side peek panel is gone — node config is now edited inline on the card, Unreal-blueprint style. Each card shows its label, intent, model, effort, prompt, and branch by default, height-capped with a gradient fade and a ghost chevron that expands to the full height when there's more (run status, errors via Callout, and an output preview); the full transcript opens the session on double-click. Gate nodes show their status, approval controls, and explainer inline too.
- 6c8f618: Workflow node and run-history polish: a running node now shows a braille spinner in its header instead of a "Running… double-click to open" line and a status badge, the config form is tighter (smaller textarea, less padding) to keep nodes compact, and run history is a read-only list of the last 5 runs with their status and timestamps — viewing a past run on the canvas is gone.
- 5ce652f: Workflow splits now run in parallel: read-only nodes (plan, review) execute concurrently while nodes that write code take the shared worktree exclusively, so a code step fanning out to several reviewers runs them all at once. Gates wait for in-flight branches to drain before pausing, and a failed branch lets its siblings finish before the run settles. The editor's status pill also opens a run history: pick any past run to view its node states and outputs on the canvas, then jump back to the latest.
- 5ce652f: Workflow runs can be retried: a failed run (including one interrupted by an app restart) gets a Retry button that re-runs only the unfinished nodes in the same worktree, keeping completed work. Review/Revise nodes are told explicitly when the worktree has no changes yet, and running a workflow from outside its project's group no longer files its sessions into the wrong group.
- 6c8f618: Workflow runs no longer look stuck while a fresh worktree installs dependencies: setup (e.g. `bun install`) runs off the critical path, node sessions appear immediately, and each node shows a "Setting up workspace" status until the tree is ready (a code node still waits so it never runs against missing deps). A crashed executor now settles the run as failed instead of leaving a phantom "Running". The Run button is disabled up front when a required prompt is empty, and run history moves from the status-pill dropdown into a collapsible table tucked above the zoom controls.
- 77d5b80: Worktree cleanup on Windows now retries when a file is briefly locked (antivirus, the search indexer), so a deleted session's worktree is removed instead of leaking.
