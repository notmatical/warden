## Findings deferred (real, but not planned yet)

Surfaced in the audit, deliberately not turned into plans in this batch:

  `agent/session_proc.rs:143-154`, `providers/codex/agent.rs:140-142`): `let _ =`
  / fire-and-forget tasks log nothing on failure, so a failed prompt-write or
  output-drain shows up only as a silently hung session. MED effort. Good
  follow-up once 002's testing pattern exists; pairs with the item below.
- **Session can stick in `running` after fast respawn** (`agent/session_proc.rs:
  ~306-366`): the `proc_id` generation guard narrows but may not fully close the

  needs a concrete trace/test before it earns a plan.

  workflow-node output** (`store/mod.rs:363`): O(n) per call; only matters for
  very long sessions feeding workflows. Low urgency.

  HIGH-risk to split; do it only after the store has characterization tests.

## Findings considered and rejected

So nobody re-audits them:

- **git `show {base}:{path}` "command injection / RCE"** — `run()` uses
  `Command::new("git")` with an args array (`git/cli.rs:11`), no shell; git bounds
  the path to the tree. Not exploitable.
- **Attachment symlink "path traversal"** (`agent/attachments.rs:71`) — by design:
  the user drops files for their own agent to read; no privilege boundary crossed.
- **`gh pr create --title/--body` "flag injection"** (`integrations/github/pr.rs:
  148`) — title/body are separate argv elements; they cannot inject flags.
- **"Mention path injection"** — speculative; mention resolve returns repo-relative
  paths the user picked. No concrete vector.
- **`setSessionPinned` "stale rollback"** (`store/slices/sessions.ts:276-281`) —
  the rollback correctly restores `prev.pinned` while preserving concurrent fields.
- **`reorderTab` "wrong position"** (`store/slices/viewport.ts:133-134`) — a
  consistent insert-before-target semantic, not a bug.
- **app-store focus listeners "leak"** (`store/app-store.ts:92-102`) — init-guarded;
  dev/HMR-only, negligible.
- **Dependency "EOL / pre-release" findings** (Vite, React, TypeScript pins) —
  applied 2024 package knowledge to this 2026 toolchain with no evidence of actual
  breakage. Unsubstantiated.
- **PR poller "extra queries on merge"** (`integrations/github/poll.rs:70-87`) —
  those queries run only on the rare merge event and are properly guarded.

Confirmed clean during the audit: SQL is fully parameterized; Linear API key is in

JSON-RPC reader is panic-safe.
